Types of Staircases (8 main types):
  1. Straight flight staircase:
     Single straight run from one floor to another. No change in direction.
     Simplest and most economical. Used in factories, warehouses.
  2. Dog-legged staircase:
     Two flights running in opposite directions (180 deg turn) with a
     common landing between them. Most common in residential buildings.
     No open well between the flights.
  3. Open-well staircase:
     Two or more flights arranged around a rectangular open well/void.
     Provides natural light and ventilation. Used in public buildings.
  4. Spiral / Geometrical staircase:
     Helical shape around a central column or open well.
     Space-saving, decorative. Used in large halls and showrooms.
  5. Bifurcated (Imperial) staircase:
     One wide flight at the bottom splits into two narrower flights
     at a landing going in opposite directions. Used in public buildings.
  6. Quarter-turn staircase:
     Changes direction by 90 deg at a landing or by using winder steps.
  7. Three-quarter turn staircase:
     Direction changes by 270 deg. Used where floor area is limited.
  8. Circular staircase:
     Continuous curved form. Grand entrances, decorative purposes.
  Components: Flight, Landing, Tread (horizontal), Rise (vertical),
  Nosing, Soffit, Baluster, Handrail, Newel post, Going.
##END##

Types of Arches (by shape):
  1. Semicircular arch: rise = half the span. Strong, widely used in masonry.
  2. Segmental arch: rise < half span; a segment of a circle. Most common
     for door and window openings (flatter appearance).
  3. Pointed / Gothic arch: two curves meeting at a point at the crown.
     Used in Gothic architecture, churches.
  4. Flat / Jack arch: nearly flat; span is small; used for small openings.
  5. Elliptical arch: forms an ellipse; pleasing appearance; used in bridges.
  6. Parabolic arch: parabolic shape; efficient for self-weight loads; bridges.
  7. Horseshoe arch: more than a semicircle (>180 deg); Islamic architecture.
  Parts of an arch: Intrados (soffit), Extrados (back), Crown, Springing line,
  Keystone, Voussoir, Skewback, Haunch, Abutment, Spandrel.
##END##

#TOPIC=civil
#KEYS=bridge types classification span
Types of Bridges (by structural form):
  1. Beam / Girder bridge: simplest form; beam rests on two supports.
     Used for short to medium spans.
  2. Truss bridge: triangulated framework of members; efficient for
     medium spans (50-150 m). Example: Warren truss, Pratt truss.
  3. Arch bridge: load transferred to abutments by arch action.
     Suitable for rocky foundations. Spans up to 500 m.
  4. Suspension bridge: deck hung from cables anchored at towers.
     Longest spans (>500 m). Example: Golden Gate Bridge.
  5. Cable-stayed bridge: deck supported directly by cables from pylons.
     Spans 200-1000 m. Modern appearance.
  6. Cantilever bridge: overhanging beams counterbalanced. Example: Howrah.
  7. Moveable bridges: bascule (drawbridge), swing, vertical-lift.
  Classification by use: Road bridge, Railway bridge, Aqueduct, Flyover.
  Classification by span: Culvert (<6m), Minor (<30m), Major (>30m).
##END##

Types of Roofs:
  FLAT ROOFS:
    Flat / Terrace roof: slope < 10 deg. RCC/brick used. Common in hot
    dry climates. Provides usable terrace space.
  PITCHED ROOFS (slope > 10 deg):
    1. Lean-to (shed) roof: single slope; simplest pitched roof.
    2. Gable roof: two slopes meeting at a central ridge; triangular gable ends.
    3. Hip roof: all four sides slope downward; no vertical gable ends.
    4. Gambrel roof: two slopes on each side; lower slope steeper.
    5. Mansard roof: double slope on all four sides; extra usable floor space.
  TRUSSED ROOFS:
    King post truss, Queen post truss, Fink truss, Howe truss.
    Used for large spans in factories, halls, railway stations.
  SHELL ROOFS:
    Barrel vault, Dome, Hyperbolic paraboloid (hypar), Folded plate.
    Used for large column-free areas; aesthetically pleasing.
##END##

#TOPIC=civil
#KEYS=dam types classification gravity arch earthen
Types of Dams (by construction material and structural action):
  1. Gravity dam: relies on its own weight to resist water pressure.
     Made of concrete or masonry. Example: Bhakra Dam.
  2. Arch dam: curved structure transferring water pressure to valley walls.
     Requires strong rocky foundations on both sides.
     Example: Hoover Dam, Idukki Dam.
  3. Earthen / Embankment dam: compacted earth/soil fill.
     Largest category by number. Example: Tehri Dam.
  4. Rockfill dam: rock fragments as the main fill material with
     impervious clay core or concrete facing for watertightness.
  5. Buttress dam: sloping slab supported by buttresses. Less material
     than gravity dam. Types: Flat slab, Multiple arch, Massive head.
  6. Composite dam: combination of above types.
  Components: Crest, Free board, Spillway, Gallery, Toe, Heel,
  Diversion tunnel, Drainage blanket.
##END##

#TOPIC=civil
#KEYS=flexible rigid pavement road types difference
Flexible vs Rigid Pavement:
  FLEXIBLE PAVEMENT (Bituminous/Asphalt):
    Layers: Subgrade -> Sub-base -> Base course -> Binder course -> Wearing course.
    Load transferred by grain-to-grain contact (interlocking).
    Low flexural strength; distributes load over a wider area.
    Design life: 10-15 years. Easy to repair; cheaper initial cost.
    Failure: Rutting, cracking, potholes.
  RIGID PAVEMENT (Cement Concrete):
    Layers: Subgrade -> Sub-base -> PCC slab.
    PCC slab (M30-M40 grade) has high flexural strength.
    Load carried by slab action (beam action).
    Design life: 30-40 years. Higher initial cost; low maintenance.
    Joints provided: Expansion joint, Contraction joint, Construction joint.
    Failure: Slab cracking, pumping, faulting.
  IRC (Indian Roads Congress) code governs road design in India.
##END##

#TOPIC=civil
#KEYS=prestressed concrete pre-tensioning post-tensioning
Prestressed Concrete (PSC):
  Concrete with high-tensile steel tendons stressed before loading
  to counteract tensile stresses due to bending.
  WHY: Concrete is weak in tension (~3 MPa); PSC eliminates tension.
  PRE-TENSIONING:
    Tendons stressed BEFORE concrete is cast. Tendons bonded to concrete.
    Done in factory (precast). Bond transfers prestress to concrete.
    Used for: Railway sleepers, poles, small beams, bridge girders.
  POST-TENSIONING:
    Tendons stressed AFTER concrete has hardened. Ducts cast in concrete.
    Tendons inserted, stressed, then anchored at ends.
    Used for: Large bridges, tanks, slabs, nuclear structures.
  High-tensile steel: wires (2-7mm), strands (7-wire), bars (25-40mm).
  Grade: Fe1600, Fe1800, Fe1960.
  Advantages: Larger spans, thinner sections, no cracks under service load.
##END##

#KEYS=formwork shuttering types timber steel aluminum
Types of Formwork (Shuttering):
  Formwork: temporary structure that holds wet concrete in place until
  it gains sufficient strength.
  1. Timber formwork: traditional; flexible and easy to cut; low initial cost;
     limited reuse (5-10 times). Most common for small works.
  2. Plywood formwork: better surface finish; lighter than solid timber.
  3. Steel formwork: high initial cost; can be reused 100+ times;
     smooth concrete finish; used for large repetitive structures.
  4. Aluminium formwork (MIVAN technology): very light, high reuse,
     used for mass housing; produces very smooth surfaces.
  5. Plastic/GRP formwork: lightweight, rust-free, gives good finish.
  6. Slip formwork: continuously moved upward as concrete sets;
     used for silos, shafts, chimneys (continuous vertical structures).
  7. Jump formwork: hydraulically lifted; used for tall buildings/dams.
  Striking (removal) time per IS 456:
    Columns/walls: 24-48 hours. Slabs: 14 days. Beams: 14-21 days.
##END##

#TOPIC=civil
#KEYS=types beams simply supported cantilever fixed continuous
Types of Beams (by end condition / support):
  1. Simply supported beam: rests on two supports; free to rotate at ends.
     BM at supports = 0; max BM at centre for UDL = wL^2/8.
  2. Cantilever beam: one end fixed, other end free.
     Max BM at fixed end = wL^2/2 (for UDL).
  3. Fixed beam (built-in/encastre): both ends fully fixed.
     Develops fixing moments at both ends; less deflection than SS beam.
  4. Continuous beam: extends over more than two supports.
     Economical; used in bridges and building floors.
  5. Propped cantilever: cantilever with a prop/support at free end.
  6. Overhanging beam: extends beyond one or both supports.
  Types by cross-section: T-beam, L-beam, Box beam, I-section, Hollow.
  Types by material: RCC beam, Steel beam, Timber beam, Composite beam.
##END##

#KEYS=slab types one way two way flat plate waffle ribbed
Types of Slabs:
  1. One-way slab: Ly/Lx > 2. Load transfers along shorter span only.
     Reinforcement provided along shorter span (main steel).
     Distribution steel along longer span.
  2. Two-way slab: Ly/Lx <= 2. Load transfers in both directions.
     Reinforcement in both directions; more efficient; used for square panels.
  3. Flat plate slab: RCC slab supported directly on columns (no beams/drop panels).
     Simple formwork; minimum depth; used in apartments.
  4. Flat slab: slab directly on columns WITH drop panels / column capitals.
     Better shear resistance than flat plate.
  5. Waffle (grid) slab: two-way ribbed slab; reduces dead weight;
     used for large column-free spans in auditoriums/halls.
  6. Hollow core slab: precast; voids reduce weight; used in offices/parking.
  7. Composite slab: steel deck with concrete topping; used in steel buildings.
##END##

#KEYS=column types shape braced unbraced short long
Types of Columns:
  By length/slenderness:
    Short column: leff/d < 12 (or leff/r < 45). Fails by crushing (material failure).
    Long/Slender column: leff/d >= 12. Fails by buckling (Euler's formula).
    leff = effective length; r = radius of gyration.
  By shape of cross-section:
    Square, Rectangular, Circular (most common).
    L-shaped, T-shaped, Cross-shaped (special boundary columns).
  By lateral bracing:
    Braced column: lateral loads taken by shear walls/bracings; column carries axial load only.
    Unbraced column: must resist lateral loads itself; larger moments.
  By position in plan:
    Corner column, Edge column, Interior column.
  By material: RCC column, Steel column (H-section, hollow section), Composite column.
  Effective length (leff) per IS 456:
    Both ends fixed: 0.5L.  One fixed, one pinned: 0.7L.  Both pinned: L.  One fixed, one free: 2L.
##END##

Types of Scaffolding:
  1. Single (brick layer's) scaffolding: one row of vertical standards (uprights)
     near the wall. Ledgers connect standards; putlogs rest on ledgers and wall.
     Used for brick masonry work.
  2. Double (mason's/independent) scaffolding: two rows of uprights; independent
     of wall. Used for stone masonry (cannot make holes in stone wall).
  3. Cantilever (needle) scaffolding: needles project from walls/openings.
     Used when ground does not allow standards (traffic, drains).
  4. Suspended scaffolding: platform hung from roof by ropes/chains.
     Used for painting, repairs on tall buildings.
  5. Trestle scaffolding: platform supported on movable tripods/ladders.
     Used indoors for heights up to 5m.
  6. Steel tube (tubular) scaffolding: steel tubes and couplers.
     Strong, durable, flexible for any shape. Most common today.
  7. Patented scaffolding: readymade proprietary systems with special
     couplings and frames. Fast to erect and dismantle.
##END##

Types of Brick Masonry Bonds:
  A bond is the arrangement of bricks in successive courses to ensure
  overlapping of vertical joints (no continuous vertical joints).
  1. Stretcher bond: only stretchers visible in face; used for half-brick
     partition walls. Weakest bond.
  2. Header bond: only headers visible; used for curved walls and piers.
  3. English bond: alternate courses of stretchers and headers.
     Strongest bond. Most commonly used in India.
  4. Flemish bond: each course has alternate stretchers and headers.
     Stronger appearance but structurally weaker than English bond.
     Single Flemish: English bond at back, Flemish at face.
     Double Flemish: Flemish bond throughout thickness.
  5. Dutch bond (modified English): corners of English bond are stronger.
  6. Zigzag bond: bricks laid in zigzag pattern; used for paving.
  7. Garden wall bond: variation of English/Flemish for thin one-brick walls.
##END##

#KEYS=pointing types masonry weathered flush recessed tuck
Types of Pointing (Finishing of Mortar Joints):
  Pointing = finishing the mortar joints in brickwork/stonework after
  raking out the old/fresh mortar to a depth of 12-20mm.
  1. Flush pointing: mortar pressed flush with the face. Most common.
     Gives a neat appearance; not very durable.
  2. Recessed (sunk) pointing: mortar pressed back 5mm from face.
     Creates shadow lines; modern appearance; less durable.
  3. Weathered (struck) pointing: top of joint pressed in, bottom projects.
     Sheds water well; durable; used for external walls.
  4. V-groove pointing: V-shaped groove formed in joint. Decorative.
  5. Tuck pointing: mortar filled flush; then a narrow ridge of lime putty
     (tuck) pressed in centre. Gives appearance of fine joints.
  6. Beaded pointing: half-round bead formed along joint. Decorative.
  Mortar for pointing: 1:3 cement:sand or lime mortar depending on exposure.
##END##

#TOPIC=civil
#KEYS=mortar types lime cement gauged surkhi
Types of Mortar:
  Mortar = binding material made by mixing a cementing agent + fine aggregate + water.
  1. Lime mortar: slaked lime + sand (1:2 or 1:3). Slow setting; flexible;
     good for old masonry repairs. Not suitable for wet locations.
  2. Cement mortar: OPC + sand. Common ratios: 1:3 (rich), 1:6 (lean).
     Strong, durable, impermeable. Widely used.
  3. Gauged mortar: lime + cement + sand. Combines workability of lime
     with strength of cement. Common: 1:1:6 (cement:lime:sand).
  4. Surkhi mortar: burnt brick powder (surkhi) + lime. Hydraulic;
     sets under water; used for foundations, cisterns.
  5. Gypsum mortar: gypsum plaster + water. Sets very fast; interior use only.
  6. Mud mortar: clay soil + water (+ sometimes lime/cowdung).
     Cheapest; used for rural/temporary construction.
  Grade of mortar per IS 2250: MM0.7, MM1.5, MM3.0, MM5.0, MM7.5, MM10.
##END##

#KEYS=plastering types cement lime gypsum
Types of Plastering:
  Plastering = applying mortar coat on wall/ceiling surfaces for:
  (a) protection from weather, (b) smooth surface, (c) decoration.
  1. Lime plaster: slaked lime + sand. 3 coats: rendering, floating, setting.
     Smooth finish; flexible; used for old buildings.
  2. Cement plaster: OPC + sand (1:4 or 1:6). Durable, hard, waterproof.
     Widely used for external and internal walls.
  3. Gypsum plaster: quick-setting, smooth interior finish; no shrinkage cracks.
     Cannot be used in damp locations.
  4. Mud plaster: clay + straw/grass fiber. Used in rural areas.
  5. Stucco plaster: cement + sand + lime. 3 coats; decorative exterior finish.
  6. Waterproof plaster: cement + sand + waterproofing compound.
     Used for bathrooms, water tanks, basements.
  Coats: Scratch coat (first, 12mm), Floating coat (8mm), Finishing coat (3mm).
##END##

#TOPIC=civil
#KEYS=admixtures types plasticizer superplasticizer retarder accelerator
Types of Admixtures (IS 9103):
  Admixtures are materials added to concrete (other than cement, water,
  aggregate) to modify properties of fresh or hardened concrete.
  1. Plasticizers (Water Reducers): reduce water content by 5-10% for
     same workability. Increase strength. Example: Lignosulfonates.
  2. Superplasticizers (High Range Water Reducers): reduce water by
     15-30%. Produce flowing/self-compacting concrete. Example: Naphthalene,
     Polycarboxylate ether (PCE).
  3. Retarders: slow down hydration; used in hot weather and for large pours.
     Example: Calcium lignosulfonate, Sugar, Citric acid.
  4. Accelerators: speed up hydration; early strength gain; used in cold
     weather and precast. Example: Calcium chloride (CaCl2).
  5. Air-entraining agents: introduce tiny air bubbles (3-6%);
     improve freeze-thaw resistance. Example: Vinsol resin.
  6. Waterproofing agents: reduce permeability. Integral waterproofing.
  7. Mineral admixtures (supplementary cementitious materials):
     Fly ash (class C, F), GGBS, Silica fume, Metakaolin.
##END##

#TOPIC=civil
#KEYS=aggregate tests impact crushing abrasion elongation
Tests on Coarse Aggregate:
  1. Aggregate Impact Value (AIV) test (IS 2386 Part IV):
     Measures resistance to sudden impact/shock loads.
     Aggregate in cylinder struck 15 blows by 14kg hammer.
     AIV = (fines passing 2.36mm / original mass) x 100%
     Limits: Roads < 30%; concrete < 45%; wearing surface < 30%.
  2. Aggregate Crushing Value (ACV) test (IS 2386 Part IV):
     Measures resistance to crushing under gradually applied load.
     ACV = (fines / original) x 100%. Limit < 30% for roads, < 45% for concrete.
  3. Los Angeles Abrasion test (IS 2386 Part IV):
     Resistance to wear/abrasion. Aggregate + steel balls in rotating drum.
     Limit < 30% for wearing surfaces, < 50% for sub-base material.
  4. Elongation Index: % of elongated particles (length > 1.8 x mean size).
     Limit < 15% for good quality aggregate.
  5. Flakiness Index: % of flat particles (thickness < 0.6 x mean size).
     Limit < 15%. High flakiness reduces workability and strength.
  6. Specific gravity: coarse aggregate = 2.6-2.9.
     Water absorption < 2% for good aggregate.
##END##

#TOPIC=civil
#KEYS=workability tests vee-bee compaction factor
Workability Tests for Concrete:
  1. Slump test (IS 1199): most common; suitable for medium workability.
     Not suitable for very dry or very wet mixes.
  2. Vee-Bee (Vibration) test: suitable for very dry, low workability mixes.
     Concrete in slump cone on vibrating table; time (seconds) to compact = Vee-Bee degree.
     Lower time = more workable. Used for precast and road concrete.
  3. Compaction Factor test: measures degree of compaction relative to fully
     compacted concrete. Two hoppers; concrete falls into a cylinder.
     CF = mass of partially compacted / mass of fully compacted.
     Range: 0.7 (low) to 0.95 (high workability).
     More sensitive than slump test for low workability mixes.
  4. Flow table test: measures spread diameter of concrete on a flow table.
     Used for self-compacting concrete and high workability mixes.
  5. Kelly Ball test: penetration of a heavy ball; suitable for field use.
##END##

#TOPIC=civil
#KEYS=joints concrete expansion construction contraction isolation
Types of Joints in Concrete Structures:
  1. Expansion joint: allows thermal expansion and contraction of structure.
     Full separation between two sections; filled with compressible filler.
     Provided every 30-45m in pavements, every 30m in buildings.
  2. Construction (cold) joint: formed when concreting is stopped and
     resumed later. Not a designed joint; must be properly prepared
     (roughened, cleaned, bonding agent applied) before fresh concrete.
  3. Contraction (control) joint: induces cracking at a controlled location.
     Sawn to 1/4 slab depth. Provided every 4-6m in pavements.
  4. Isolation joint: separates structural slab from walls, columns, footings;
     allows differential movement without cracking.
  5. Day joint: construction joint between concreting operations done on
     different days; formed at end of day's pour.
##END##

Types of Piles (Foundation):
  By material:
    Timber piles: cheap, used in marine works; treated against decay.
    Concrete piles: most common; precast (driven) or cast-in-situ (bored).
    Steel piles: H-piles, pipe piles; high load capacity; costly.
    Composite piles: combination of materials.
  By load transfer mechanism:
    End-bearing piles: tip rests on hard stratum; load via tip resistance.
    Friction piles: load transferred via skin friction along shaft.
    Combination: both friction and end bearing.
  By method of installation:
    Driven piles: hammered into ground (displacement piles).
    Bored (cast-in-situ) piles: hole bored, then concrete cast. No vibration.
    Under-reamed piles: bored pile with one or more bulbs at base;
      used in expansive soils (black cotton soil); IS 2911.
  Pile cap: connects tops of piles into a group; transfers column load to piles.
##END##

#TOPIC=civil
#KEYS=weir barrage difference irrigation diversion
Weir vs Barrage:
  WEIR:
    A structure built across a river to raise the water level on the
    upstream side and allow controlled diversion into canals.
    No gates or with only limited gated openings.
    Water flows over the crest of the weir (overflows).
    Permanent structure; simple; low cost.
    Cannot regulate water level effectively during floods.
    Examples: Okhla Weir (Delhi), Mettur Weir (Tamil Nadu).
  BARRAGE:
    A low dam across a river with a series of gates extending the full
    length of the structure. Gates fully regulate water level.
    Water flows THROUGH the gates (not over).
    More flexible water level control; handles floods better.
    More expensive; complex construction.
    Examples: Farakka Barrage, Bhimgoda Barrage.
  Both are diversion structures (do not store water like dams).
##END##

#TOPIC=civil
#KEYS=CPM PERT network analysis difference project management
CPM vs PERT (Network Analysis):
  Both are project scheduling techniques using network diagrams.
  CPM (Critical Path Method):
    Deterministic: single time estimate per activity.
    Used when durations are well-known (construction projects).
    Finds the CRITICAL PATH: longest path from start to finish.
    Activities on critical path: any delay delays the whole project.
    Float: difference between latest and earliest start time.
  PERT (Programme Evaluation and Review Technique):
    Probabilistic: THREE time estimates per activity:
      Optimistic (to), Most likely (tm), Pessimistic (tp).
      Expected time: te = (to + 4tm + tp) / 6.
    Used for R&D, new projects with uncertain durations.
    Calculates probability of completing by a given date.
  Both use: Activity, Event, Dummy activity, Float, Critical path.
  Difference: CPM = deterministic + activity-on-arrow/node;
              PERT = probabilistic + activity-on-arrow.
##END##

#TOPIC=civil
#KEYS=qualities good brick properties standard
Qualities of a Good Brick:
  1. Shape and size: uniform rectangular shape with sharp edges and
     plane surfaces. Standard size: 190x90x90 mm (IS 1077).
  2. Color: uniform deep red, cherry, or copper color indicating good burning.
  3. Hardness: should resist scratching by a fingernail.
  4. Compressive strength: min 3.5 N/mm^2 (Class III) to 10.5 N/mm^2 (Class I).
  5. Water absorption: should not exceed 20% of dry weight after 24 hours
     immersion (Class I) and 22% (Class II).
  6. Efflorescence: white patches from soluble salts. Should be 'nil' to
     'slight' for good quality bricks (IS 3495 test).
  7. Soundness: when two bricks are struck together, should produce a
     clear ringing metallic sound (not dull or hollow).
  8. Thermal insulation: low thermal conductivity; keeps buildings cool.
  9. Durability: should withstand weather, frost, and rain without crumbling.
  10. Warpage: should be minimal (< 3mm for a 190mm brick).
##END##

#TOPIC=civil
#KEYS=surveying levelling types differential profile reciprocal
Types of Levelling in Surveying:
  Levelling determines relative heights (elevations) of different points.
  Terms: Benchmark (BM), Back Sight (BS), Fore Sight (FS), Intermediate Sight (IS),
         Height of Instrument (HI), Reduced Level (RL).
  1. Simple levelling: instrument set up at one point; levels two points.
  2. Differential (fly) levelling: instrument moved multiple times to
     find difference in elevation between distant points.
  3. Profile levelling: elevations along a fixed line (road/pipeline centreline).
     Produces longitudinal section.
  4. Cross-section levelling: elevations at right angles to profile line.
  5. Reciprocal levelling: level measured from both ends to eliminate
     errors of collimation and curvature/refraction. Used for wide rivers.
  6. Precise levelling: high-accuracy; used for benchmarks; invar staff.
  Errors: Error of collimation, Curvature of earth (0.0785 D^2 metres),
  Atmospheric refraction (0.0112 D^2 metres, opposes curvature).
##END##

#TOPIC=civil
#KEYS=types walls load bearing partition cavity shear
Types of Walls in Building Construction:
  1. Load-bearing wall: carries loads from slabs, beams, roofs down to
     foundation. Made of brick/stone/concrete. Cannot be removed easily.
  2. Non-load-bearing (partition) wall: divides interior space only;
     carries no structural load. Can be removed/relocated.
     Materials: brick, AAC block, glass, gypsum board, plywood.
  3. Cavity wall: two leaves (wythes) of masonry with a 50mm air gap.
     Excellent thermal and moisture insulation. Used in coastal areas.
  4. Shear wall: vertical RCC wall resisting lateral (wind/earthquake) loads.
     Located at building core or perimeter. Essential in high-rise buildings.
  5. Retaining wall: resists lateral earth/soil pressure on sloped ground.
     Types: gravity, cantilever, counterfort.
  6. Parapet wall: low wall along edge of roof/balcony. Provides safety.
  7. Curtain wall: non-load-bearing exterior wall hung from structural frame.
     Glass/aluminium curtain walls in modern high-rise buildings.
##END##

#TOPIC=civil

Concrete Mix Design (IS 10262):
  PURPOSE: Determine the proportions of cement, water, FA, CA to achieve
  target strength, workability, and durability.
  NOMINAL MIX (IS 456 Table 9): for M5 to M20 grade only.

Archimedes' Principle:
  When a body is partially or fully submerged in a fluid, it experiences
  an upward buoyant force equal to the weight of fluid displaced.
  Buoyant force (Fb) = rho_fluid * V_submerged * g
  Conditions:
    If Fb > Weight of body: body FLOATS (less dense than fluid).
    If Fb = Weight of body: body is in neutral equilibrium (suspended).
    If Fb < Weight of body: body SINKS.
  Law of flotation: a floating body displaces fluid equal to its own WEIGHT.
  Applications: ship design, hot air balloons, submarines, hydrometers.
  Relative density (specific gravity) = density of substance / density of water.
##END##

#TOPIC=physics
#KEYS=bernoulli theorem principle fluid flow pressure velocity
Bernoulli's Theorem (Principle):
  For a steady, incompressible, non-viscous fluid flowing along a streamline:
  P + (1/2)*rho*v^2 + rho*g*h = constant
  P = pressure, v = velocity, h = height, rho = density.
  Meaning: as fluid velocity increases, pressure decreases (and vice versa).
  Applications:
    Venturimeter: measures flow rate; velocity increases in throat -> pressure drop.
    Aerofoil/aircraft wing: faster flow over curved top -> lower pressure -> lift.
    Spray gun/atomizer: fast air stream creates low pressure; liquid drawn up.
    Pitot tube: measures fluid velocity by comparing static and stagnation pressure.
  Limitations: does not apply to viscous fluids, compressible gases at high speeds,
  or turbulent flow.
##END##

#KEYS=faraday law electromagnetic induction EMF
Faraday's Laws of Electromagnetic Induction:
  First Law: whenever the magnetic flux through a coil changes, an EMF
  (electromotive force) is induced in the coil.
  Second Law: the magnitude of induced EMF is proportional to the rate of
  change of magnetic flux.
  EMF = -N * d(phi)/dt
  N = number of turns, phi = magnetic flux = B * A * cos(theta).
  Negative sign: Lenz's Law (induced current opposes the change in flux).
  Applications: electric generators, transformers, induction motors,
  dynamic microphones, inductive sensors.
  Mutual induction: EMF induced in coil B due to changing current in coil A.
  Self-induction: EMF induced in coil due to its own changing current.
  Self-inductance L: EMF = -L * dI/dt [Henry].
##END##

#TOPIC=physics
#KEYS=photoelectric effect Einstein photon work function
Photoelectric Effect:
  When light (photons) of sufficient frequency strikes a metal surface,
  electrons are emitted from the surface.
  Einstein's explanation (1905, Nobel Prize): light consists of photons.
  Energy of photon: E = hf (h = Planck's constant, f = frequency).
  Work function (phi_0): minimum energy needed to eject an electron from the surface.
  Threshold frequency: f_0 = phi_0 / h. Below this, no emission regardless of intensity.
  Einstein's equation: KE_max = hf - phi_0
  Observations:
    - No emission below threshold frequency, even with high intensity.
    - Above threshold: emission is instantaneous.
    - Increasing intensity increases number of photons (current), NOT their energy.
    - Increasing frequency increases kinetic energy of emitted electrons.
  Application: solar cells, photodiodes, CCD cameras, photomultiplier tubes.

#TOPIC=physics
#KEYS=Lenz law magnetic induction direction oppose
Lenz's Law:
  The direction of the induced current (or EMF) is always such that it
  opposes the change in magnetic flux that caused it.
  This is the negative sign in Faraday's law: EMF = -N * d(phi)/dt.
  Consequence: induced current creates a magnetic field that:
    - Opposes increase in flux (when flux increases, induced B opposes external B).
    - Opposes decrease in flux (when flux decreases, induced B tries to maintain it).
  Lenz's law is a statement of conservation of energy: work must be done
  against the opposing force to maintain the flux change.
  Application: eddy current braking in trains and roller coasters,
  electromagnetic damping in galvanometers, induction cookers.
##END##

Differential Equations:

Fourier Series:
  Any periodic function f(x) of period 2L can be expressed as an
  infinite sum of sines and cosines:
  f(x) = a0/2 + SUM[n=1 to inf] (an*cos(n*pi*x/L) + bn*sin(n*pi*x/L))
  Coefficients: a0 = (1/L)*INT[-L to L] f(x)dx
                an = (1/L)*INT[-L to L] f(x)*cos(n*pi*x/L)dx
                bn = (1/L)*INT[-L to L] f(x)*sin(n*pi*x/L)dx
  Dirichlet conditions: function must be periodic, bounded, piecewise continuous.
  FOURIER TRANSFORM (for non-periodic signals):
  F(omega) = INT[-inf to inf] f(t) * e^(-i*omega*t) dt
  Converts signal from time domain to frequency domain.
  Applications: signal processing, image compression (JPEG), audio processing,
  solving PDEs, vibration analysis.

Linear Programming (LPP):
  Optimization of a LINEAR objective function subject to LINEAR constraints.
  Standard form: Maximize/Minimize Z = c1*x1 + c2*x2 + ... + cn*xn
  Subject to: a11*x1 + a12*x2 + ... <= b1 (constraints), xi >= 0.
  GRAPHICAL METHOD (2 variables):
    1. Plot constraint lines. 2. Identify feasible region (convex polygon).
    3. Optimal solution at a corner (vertex) of feasible region.

    Moves along edges of feasible polytope to reach optimal corner.
  Key concepts:

    Optimal solution: best feasible solution (min or max of objective).
    Degenerate solution: basic feasible solution with a zero basic variable.
    Unbounded solution: objective can increase/decrease indefinitely.
    Infeasible: no solution satisfies all constraints.
  Applications: production planning, diet problem, transportation, scheduling.

Model Evaluation Metrics:
  CONFUSION MATRIX (binary classification):
    TP=True Positive, TN=True Negative, FP=False Positive, FN=False Negative.
  ACCURACY:  (TP+TN) / (TP+TN+FP+FN). Misleading for imbalanced classes.
  PRECISION: TP / (TP+FP). Of predicted positives, how many are correct?
             "How precise is the model when it predicts positive?"
  RECALL (Sensitivity): TP / (TP+FN). Of actual positives, how many found?
             "How many actual positives did the model find?"

             Used when both precision and recall matter equally.
  Specificity: TN / (TN+FP).
  ROC Curve: plot of TPR vs FPR at different thresholds.
  AUC (Area Under ROC Curve): 1.0 = perfect, 0.5 = random.
  For multi-class: Macro-average (equal weight per class), Micro-average.
  When to use:
    Precision: spam detection (don't want false alarms).
    Recall: cancer detection (don't miss any positives).
    F1: information retrieval, NLP tasks.

Principal Component Analysis (PCA):
  Unsupervised technique to reduce dimensionality of data by finding
  directions (principal components) of maximum variance.
  STEPS:
    1. Standardize data (zero mean, unit variance).
    2. Compute covariance matrix.
    3. Compute eigenvectors and eigenvalues of covariance matrix.
    4. Sort eigenvectors by eigenvalue (descending).
    5. Select top k eigenvectors as principal components.
    6. Project data onto k-dimensional subspace.
  Principal components are ORTHOGONAL to each other.
  Explained variance ratio: how much variance each component captures.
  Scree plot: plot eigenvalues; look for 'elbow' to choose k.
  USES:
    Data visualization (reduce to 2D/3D for plotting).
    Noise reduction. Speed up ML algorithms. Image compression.
  Limitation: only captures LINEAR relationships. For non-linear use t-SNE or UMAP.

RNN and LSTM:
  RNN (Recurrent Neural Network):
    Processes sequential data by maintaining a hidden state across time steps.
    h_t = tanh(W_h * h_{t-1} + W_x * x_t + b)

Sorting Algorithms in C:
  1. BUBBLE SORT: O(n^2) avg/worst. Repeatedly swap adjacent elements.
     for(i=0;i<n-1;i++) for(j=0;j<n-i-1;j++) if(a[j]>a[j+1]) swap.
  2. SELECTION SORT: O(n^2). Find minimum in unsorted part; swap to front.
  3. INSERTION SORT: O(n^2) worst; O(n) best (nearly sorted). Good for small n.
  4. MERGE SORT: O(n log n) always. Divide-and-conquer; stable sort.
     Split array in half, sort each half recursively, merge.
  5. QUICK SORT: O(n log n) average; O(n^2) worst (bad pivot).
     Pick pivot; partition: elements < pivot left, > pivot right; recurse.
     In-place; cache-friendly; fastest in practice.
  6. HEAP SORT: O(n log n). Build max-heap; repeatedly extract max.
     In-place but not stable.
  Stable sort: equal elements maintain original relative order (Merge, Insertion).
  qsort() in stdlib.h: built-in C sorting with comparator function.
  int cmp(const void *a, const void *b) { return (*(int*)a - *(int*)b); }
  qsort(arr, n, sizeof(int), cmp);

Enums and Unions in C:
  ENUM (enumeration):
    User-defined type assigning names to integer constants.
    enum Color { RED=0, GREEN=1, BLUE=2 };
    enum Color c = GREEN; // c == 1
    Default: first member = 0, each subsequent += 1.
    Improves code readability; type-safe alternative to #define constants.

    vs struct: struct allocates separate space for each member.

Stack and Queue Implementation in C:
  STACK (LIFO - Last In First Out):
    Using array:
      int stack[MAX]; int top = -1;
      Push: stack[++top] = val; (check top < MAX-1)
      Pop:  val = stack[top--];  (check top >= 0)
      Peek: stack[top];
    Operations: O(1). Used for: function call stack, undo, expression eval, DFS.
  QUEUE (FIFO - First In First Out):
    Using array (circular queue):
      int q[MAX]; int front=0, rear=0, count=0;
      Enqueue: q[rear] = val; rear = (rear+1)%MAX; count++;
      Dequeue: val = q[front]; front = (front+1)%MAX; count--;
    Using linked list: no size limit; dynamic.
    Operations: O(1). Used for: BFS, scheduling, print queues, buffers.
  DEQUE (Double-ended queue): insert/delete at both ends.
  PRIORITY QUEUE: element with highest priority dequeued first (use heap).
##END##

#UNIT=3
#KEYS=binary search algorithm C O(log n)
Binary Search in C:
  Searches a SORTED array for target in O(log n) time.
  Algorithm:
    1. Set low=0, high=n-1.
    2. While low <= high:
       mid = (low + high) / 2;
       if arr[mid] == target: FOUND, return mid.
       if arr[mid] < target:  low = mid + 1 (search right half).
       if arr[mid] > target:  high = mid - 1 (search left half).
    3. Return -1 if not found.
  Code:
    int binary_search(int arr[], int n, int target) {
        int low=0, high=n-1;
        while(low<=high) {
            int mid = low + (high-low)/2; // avoids overflow vs (low+high)/2
            if(arr[mid]==target) return mid;
            else if(arr[mid]<target) low=mid+1;
            else high=mid-1;
        }
        return -1;
    }
  Vs linear search: Binary=O(log n), Linear=O(n). Binary requires sorted array.
  Recursive variant: same logic, passing low/high as parameters.
##END##

Header Guards and Include Guards in C:

Nuclear Chemistry and Radioactivity:
  Radioactivity: spontaneous emission of particles/energy from unstable nuclei.
  Types of decay:
    Alpha (alpha) decay: emits He-4 nucleus (2p + 2n). Short range in air (~5cm).

  Nuclear stability: N/Z ratio; stable band; heavy nuclei need more neutrons.

Colligative Properties of Solutions:

Electronegativity and Bond Polarity:

#TOPIC=chemistry
#KEYS=equilibrium Le Chatelier Kc Kp reversible reaction
Chemical Equilibrium and Le Chatelier's Principle:
  Reversible reaction reaches equilibrium when forward rate = reverse rate.
  Equilibrium constant Kc (at constant T):
    aA + bB <-> cC + dD: Kc = [C]^c[D]^d / ([A]^a[B]^b)
    Kc > 1: products favoured; Kc < 1: reactants favoured.
  Kp = Kc * (RT)^(delta_n)  where delta_n = moles of gas products - reactants.
  LE CHATELIER'S PRINCIPLE:
    When a system at equilibrium is disturbed, it shifts to oppose the change.
    Increase concentration of reactant -> equilibrium shifts RIGHT (more products).
    Increase pressure -> shifts toward side with fewer moles of gas.
    Increase temperature -> shifts toward endothermic side (absorbs heat).
    Catalyst: increases rate of both forward and reverse reactions equally;
    does NOT change equilibrium position or Kc, only time to reach equilibrium.
  Reaction quotient Q: same form as Kc; if Q < Kc -> shifts right; if Q > Kc -> shifts left.
##END##

Serial Communication Protocols:

PWM (Pulse Width Modulation):
  A technique of encoding an analog signal as a digital square wave
  by varying the pulse width (duty cycle).
  Duty cycle = (ON time / Total period) x 100%
    0% duty cycle: always OFF (0V output average).
    50% duty cycle: half ON, half OFF.
    100% duty cycle: always ON (full voltage average).
  Average voltage: V_avg = duty_cycle/100 * V_supply.
  Generation: MCU timer comparator register vs period register.
  APPLICATIONS:
    Motor speed control: vary duty cycle -> vary average voltage -> vary speed.
    LED brightness control (dimming).
    Power supply (SMPS): DC-DC converters use PWM to regulate output.
    Servo motor control: 50Hz PWM; 1ms pulse=0 deg, 1.5ms=90 deg, 2ms=180 deg.
    Audio: Class D amplifiers use PWM at >100kHz.
  Resolution: 8-bit PWM -> 256 levels; 10-bit -> 1024 levels.
  Frequency selection: high enough to avoid flicker/audible noise; low enough for MCU.
##END##

#KEYS=Zener diode voltage regulator breakdown
Zener Diode and Voltage Regulation:
  A diode designed to operate in REVERSE BREAKDOWN region reliably.
  Zener breakdown: occurs at low reverse voltages (< 5V) due to quantum tunneling.
  Avalanche breakdown: at higher voltages due to impact ionization.
  In breakdown, voltage across Zener stays CONSTANT (Vz) despite varying current.
  VOLTAGE REGULATOR CIRCUIT:
    Vin (unregulated) -> series resistor Rs -> Zener in parallel with load.
    Rs = (Vin - Vz) / (Iz + IL)
    Output voltage = Vz (regulated). Works as long as Iz > Iz_min.
  Zener characteristics:
    Vz: zener voltage (e.g., 3.3V, 5.1V, 6.2V, 12V).
    Power rating: Pz_max = Vz * Iz_max. Must not exceed this.
  Applications:
    Voltage reference circuits.
    Over-voltage protection (clipping circuits).
    Level shifting, waveform clipping.
  Modern alternative: LM7805, LM317 IC regulators for higher current.
##END##

Power Supply Design:

  Ripple factor = Vrms_ripple / Vdc_output. Lower is better.

Line Integral:
  A line integral evaluates a function along a curve (path) in space.
  SCALAR line integral: integral of f(x,y,z) ds along curve C.
    ds = arc length element = sqrt((dx)^2+(dy)^2+(dz)^2)
  VECTOR line integral (work integral):
    W = integral_C F.dr = integral_C (P dx + Q dy + R dz)
    where F = Pi + Qj + Rk is a vector field, dr = dx i + dy j + dz k
  Physical meaning: total work done by force field F moving a particle along C.
  If C is closed curve: written as contour integral (closed loop).
  Evaluation: parametrize curve (x=x(t), y=y(t)) and substitute.
  Example: F=xi+yj, C: x=t, y=t^2, t:0->1
    W = integral_0^1 (t*1 + t^2*2t) dt = integral_0^1 (t + 2t^3) dt = 1

#KEYS=green theorem line integral double integral curl circulation
Green's Theorem:
  Relates a line integral around a CLOSED curve C to a double integral
  over the region D enclosed by C.
  Statement: oint_C (P dx + Q dy) = iint_D (dQ/dx - dP/dy) dA
  Conditions: C is a simple, closed, positively oriented (counterclockwise)
    curve; P, Q have continuous partial derivatives in D.
  Physical meaning: circulation of vector field around C = net curl over D.
  APPLICATIONS:
    1. Compute area of region D:
       A = (1/2) oint_C (x dy - y dx)
    2. Convert difficult line integrals to easier double integrals.
    3. Verify whether a field is conservative: if dQ/dx = dP/dy throughout D,
       then oint_C F.dr = 0 (irrotational field).
##END##

Surface Integral:
  Integral of a function or vector field over a 2D surface S in 3D space.
  SCALAR surface integral: iint_S f(x,y,z) dS
    dS = surface area element = |r_u x r_v| du dv (for parametric surface)
  VECTOR surface integral (flux):
    Flux = iint_S F.dS = iint_S F.n dS
    where n = unit outward normal to surface S.
  Physical meaning: total flux (flow) of vector field F through surface S.
  Evaluation methods:
    1. Parametric: r(u,v) = x(u,v)i + y(u,v)j + z(u,v)k
       dS = (r_u x r_v) du dv
    2. Explicit z=g(x,y): n dS = (-dg/dx i - dg/dy j + k) dA
  Types: open surface (part of a plane, paraboloid) or closed surface (sphere, cube).
##END##

Gauss's Divergence Theorem:
  Relates a SURFACE integral over a closed surface S to a VOLUME integral
  over the region V enclosed by S.
  Statement: oiint_S F.dS = iiint_V div(F) dV = iiint_V (dP/dx+dQ/dy+dR/dz) dV
  where F = Pi+Qj+Rk, n = outward unit normal.
  Conditions: F has continuous first partial derivatives in V; S is closed.
  Physical meaning: total flux out of a closed surface = total divergence inside.
  APPLICATIONS:
    1. Convert hard surface integrals to easier volume integrals.
    2. Verify: sphere of radius R, F=r (position vector):
       div(r)=3 => flux = 3*(4/3 pi R^3) = 4 pi R^3.
    3. Used in electrostatics (Gauss's law in integral form).
  Steps to apply:
    1. Compute div(F) = dP/dx + dQ/dy + dR/dz.
    2. Set up volume integral using suitable coordinates.
    3. Evaluate integral.
##END##

#KEYS=stokes theorem curl line integral surface integral circulation
Stokes' Theorem:
  Relates a LINE integral around a closed curve C to a SURFACE integral
  over any surface S bounded by C.
  Statement: oint_C F.dr = iint_S (curl F).dS = iint_S (curl F).n dS
  where curl F = del x F:
    = (dR/dy-dQ/dz)i - (dR/dx-dP/dz)j + (dQ/dx-dP/dy)k
  Conditions: C is a simple closed curve, S is any surface with boundary C,
    F has continuous first partial derivatives.
  Physical meaning: circulation of F around C = flux of curl(F) through S.
  Green's theorem is a 2D special case of Stokes' theorem.
  APPLICATIONS:
    1. Evaluate line integrals using surface integrals (or vice versa).
    2. If curl F = 0 everywhere, oint F.dr = 0 for all closed paths.
  Key comparison:
    Green's theorem: 2D (plane region)
    Gauss's theorem: 3D (volume and closed surface)
    Stokes' theorem: 3D (surface and its boundary curve)
##END##

Call by Value vs Call by Reference:
  CALL BY VALUE (default in C):
    - A COPY of the argument is passed to the function.
    - Changes inside function do NOT affect original variable.
    - Example: void swap(int a, int b) -- does NOT swap originals.
  CALL BY REFERENCE (using pointers):
    - Address of variable is passed using & operator.
    - Function receives a pointer; changes via * affect original.
    - Example:
        void swap(int *a, int *b) {
            int temp = *a; *a = *b; *b = temp;
        }
        swap(&x, &y);  -- now x and y are swapped
  Passing arrays: arrays are ALWAYS passed by reference in C.
    The array name decays to a pointer to its first element.
  Passing structs: by value by default (entire struct is copied);
    pass pointer to struct for efficiency and in-place modification.
  Summary:
    By value    -- safe, no side effects; overhead for large data.
    By reference -- efficient, allows modification; needs careful use.
##END##

Storage Classes in C:
  Storage class determines: SCOPE, LIFETIME, DEFAULT VALUE, STORAGE LOCATION.
  1. AUTO (automatic):

       Retains value between function calls. Initialised once to 0.
     - Global static: scope = only that translation unit (file).
     - Example: static int count = 0; -- persists between calls

Recursion in C:
  A function that calls ITSELF (directly or indirectly) is recursive.
  BASE CASE: condition where recursion stops. RECURSIVE CASE: smaller subproblem.
  FACTORIAL:
    int fact(int n) { if (n<=1) return 1; return n * fact(n-1); }
  FIBONACCI:
    int fib(int n) { if (n<=1) return n; return fib(n-1) + fib(n-2); }
  GCD (Euclidean): int gcd(int a, int b) { return b==0 ? a : gcd(b, a%b); }

Linear Ordinary Differential Equations - Introduction:
  General nth-order linear ODE: a_n*y^(n) + ... + a_1*y' + a_0*y = g(x).

    yp = Particular Integral (any particular solution of F(D)y=g(x)).
  Linear independence: n linearly independent solutions form a basis.
  Wronskian W(y1,y2,...): if W != 0, solutions are linearly independent.

Heisenberg's Uncertainty Principle:
  Impossible to simultaneously measure with precision certain conjugate pairs.
  Position-momentum: delta_x * delta_p >= h/(4*pi).
  Energy-time: delta_E * delta_t >= h/(4*pi).
  Physical basis: particle described by wave packet; narrow packet = precise position
    but uncertain momentum; spread-out packet = precise momentum but uncertain position.
  This is a fundamental law of nature, NOT a measurement limitation.
  Consequence: cannot know present exactly, so cannot predict future exactly.
  Application: explains broadening of spectral lines (excited state has finite lifetime
    delta_t, so energy has uncertainty delta_E = h/(4*pi*delta_t)).
  Metastable states have long lifetime -> narrow spectral linewidth -> used in lasers.

Time-Independent Schrodinger Wave Equation (TISE):
  Used when potential V depends only on position (not time).
  Equation: d^2psi/dx^2 + (8*pi^2*m/h^2)*(E-V)*psi = 0.

  E = total energy (kinetic + potential), V = potential energy function.
  For free particle (V=0): d^2psi/dx^2 + k^2*psi = 0, k^2 = 8pi^2*m*E/h^2.
  TISE is a postulate of quantum mechanics (not derived from first principles).
  Solutions are energy eigenstates; each corresponds to definite energy E.

Classical Free Electron Theory (Drude-Lorentz):
  Assumptions: electrons behave like gas molecules with thermal energy (3/2)kT;
    constant ionic potential; interactions ignored.
  Successes: explains Ohm's law, derives expression for conductivity sigma = ne^2*tau/m.
  Failures:

Quantum Free Electron Theory (Sommerfeld):
  Electrons follow Pauli Exclusion Principle and Fermi-Dirac statistics.
  Fermi energy EF: highest occupied energy level at absolute zero (0 K).
  At 0 K: all levels up to EF filled; all above EF empty (sharp boundary).
  Fermi factor: f(E) = 1/[1 + exp((E-EF)/kT)].
    At T=0K: f(E)=1 for E < EF; f(E)=0 for E > EF.
    At T>0K: f(EF)=0.5 always; smooth transition around EF.
  Fermi level: energy at which f(E) = 0.5 (50% probability of occupancy).
  Successes: explains specific heat, resistivity vs T, Hall effect, thermionic emission.
  Density of states g(E) proportional to sqrt(E).
  Fermi energy: EF = (h^2/8m) * (3n/pi)^(2/3).

Band Theory of Solids:

Meissner Effect:
  When a material becomes superconducting, it EXPELS all magnetic flux from its interior.
  B = 0 inside a superconductor (not just a consequence of zero resistance).
  This is called the Meissner effect (Meissner and Ochsenfeld, 1933).
  Meissner effect proves superconductors are PERFECT DIAMAGNETS (chi = -1).
  Distinction from perfect conductor: a perfect conductor would trap existing flux;
    a superconductor expels it regardless of field history.
  Expulsion is achieved by surface currents (shielding currents) that cancel B inside.
  Application: magnetic levitation (Maglev trains, magnetic bearings).
  If applied field exceeds critical field Hc, superconductivity is destroyed.

Critical Magnetic Fields and Types of Superconductors:
  Type I superconductors: single critical field Hc; abrupt transition from
    superconducting to normal state at H = Hc. Meissner effect complete below Hc.

    H > Hc2: normal state.

    phi_0 = h/(2e) = 2.07x10^-15 Wb.

BCS Theory of Superconductivity:
  BCS = Bardeen, Cooper, Schrieffer (1957); Nobel Prize 1972.
  Mechanism: two electrons near Fermi level form a COOPER PAIR via phonon-mediated
    attractive interaction.

  Energy gap 2*delta: Cooper pairs cannot be easily scattered; gap suppresses normal scattering.
  Delta(0) = 1.76 * kTc (energy gap at 0 K related to critical temperature).
  Coherence length: spatial extent of Cooper pair; typically 100-1000 nm for low-Tc.
  BCS theory explains: zero resistance, Meissner effect, isotope effect (Tc ~ M^(-1/2)).

LASER - Principles and Basic Concepts:
  LASER = Light Amplification by Stimulated Emission of Radiation.
  Properties of laser light: monochromatic (single wavelength), coherent (in phase),
    directional (very low divergence), high intensity.
  Three key processes (Einstein 1917):
    1. ABSORPTION: photon absorbed; electron jumps to higher level.
    2. SPONTANEOUS EMISSION: electron spontaneously falls back; emits photon in random direction.
    3. STIMULATED EMISSION: incoming photon triggers electron to fall; emits IDENTICAL photon
       (same frequency, phase, direction, polarization). This is the basis of laser action.
  For amplification: stimulated emission must exceed absorption.
  POPULATION INVERSION: N2 > N1 (more atoms in excited state than ground state).
    Normally N1 > N2 (Boltzmann distribution); must be achieved by pumping.
  PUMPING: external energy input to create population inversion.
    Types: optical pumping (flash lamp), electrical discharge, chemical, semiconductor junction.

Three-Level and Four-Level Laser Systems:
  Need metastable state: excited state with long lifetime (~10^-3 s vs normal ~10^-8 s).
    Long lifetime allows population to build up (population inversion achieved).
  THREE-LEVEL LASER (e.g., Ruby laser):
    Ground state E1, pump band E3, metastable state E2.
    Pumping: E1 -> E3. Fast non-radiative decay: E3 -> E2. Laser transition: E2 -> E1.
    Problem: ground state is the lower laser level; need to depopulate it.
      More than half of all atoms must be excited -> requires high pump power.
  FOUR-LEVEL LASER (e.g., He-Ne, Nd:YAG):
    Levels: E0 (ground), E1 (pump band), E2 (upper laser level), E3 (lower laser level).
    Lower laser level E3 is above ground and quickly decays to E0.
    Advantage: E3 always nearly empty -> population inversion achieved more easily.

Ruby Laser:
  Type: Solid-state, three-level laser.
  Active medium: Ruby (Al2O3 crystal doped with 0.05% Cr^3+ ions).

Optical Fiber - Principle and Construction:
  Optical fiber transmits light by TOTAL INTERNAL REFLECTION (TIR).
  TIR: occurs when light travels from denser medium (n1) to rarer medium (n2),
    and angle of incidence exceeds critical angle theta_c = arcsin(n2/n1).
  Structure: Core (high refractive index n1), Cladding (lower index n2), Jacket.
  Light in core hits core-cladding boundary; if angle > theta_c, TIR occurs; light stays in core.
  Acceptance angle theta_0: maximum angle at which light entering fiber end is captured.
  Numerical Aperture (NA): NA = sqrt(n1^2 - n2^2) = n0*sin(theta_0).
  For air (n0=1): NA = sin(theta_0) = sqrt(n1^2 - n2^2).
  Higher NA -> accepts light from wider cone -> more light captured.
  Typical values: n1 = 1.5, n2 = 1.48 for step-index fiber.
##END##

Types of Optical Fibers and Attenuation:
  SINGLE-MODE FIBER:
    Core diameter ~8-10 um; only one mode propagates.
    No modal dispersion; suitable for long-distance, high-bandwidth applications.
    Used in telecommunications, submarine cables.
  MULTIMODE STEP-INDEX FIBER:
    Core diameter ~50-200 um; many modes propagate; abrupt index change at boundary.
    Large NA; easy to couple light; significant modal dispersion (different paths = different arrival times).
    Used for short-distance, lower bandwidth.
  MULTIMODE GRADED-INDEX FIBER:
    Core refractive index gradually decreases from center to edge.
    Different modes travel at different speeds but similar times -> less modal dispersion.
    Better bandwidth than step-index multimode.
  ATTENUATION: loss of signal intensity with distance; measured in dB/km.
    Sources: absorption (OH- ions, infrared), scattering (Rayleigh), bending losses.
    Minimum attenuation in silica fiber: ~0.2 dB/km at 1550 nm wavelength.

Bubble Sort in C:
  Bubble sort compares adjacent elements and swaps them if they are in wrong order.
  After each pass, the LARGEST unsorted element "bubbles up" to its correct position.
  Algorithm:
    for (i=0; i<n-1; i++)              -- n-1 passes
        for (j=0; j<n-1-i; j++)        -- each pass reduces inner range
            if (a[j] > a[j+1]) {
                temp=a[j]; a[j]=a[j+1]; a[j+1]=temp;  -- swap
            }
  Time complexity: O(n^2) worst and average case.
  Space complexity: O(1) (in-place sorting).
  After pass 1: largest element at last position.
  After pass 2: second largest at second-last. And so on.
  Optimization: if no swaps in a pass, array is already sorted — break early.

Isotope Effect in Superconductivity:
  Tc is proportional to M^(-1/2) where M = isotopic mass of the element.
  Heavier isotope -> lower Tc; lighter isotope -> higher Tc.
  Example: Mercury isotopes show different Tc values; heavier Hg -> lower Tc.
  Significance: proves that lattice vibrations (phonons) are involved in superconductivity.
  This was a key clue that led to BCS theory: phonons mediate Cooper pair formation.
  Transition metals that don't show isotope effect (Fe, Pd) are not superconductors.

London Equations and Penetration Depth:
  F. and H. London (1935) derived two equations explaining Meissner effect.
  First London equation: dJ_s/dt = (n_s*e^2/m) * E  (supercurrent accelerates in E field)
  Second London equation: curl(J_s) = -(n_s*e^2/m) * B  (explains Meissner effect)
  Penetration depth lambda_L: magnetic field does NOT drop to zero instantly at surface.
    B(x) = B_0 * exp(-x/lambda_L); field decays exponentially inside superconductor.
  lambda_L = sqrt(m / mu_0*n_s*e^2); typically 50-500 nm for low-Tc superconductors.
  For x >> lambda_L: B -> 0 (bulk of superconductor is field-free as Meissner effect shows).
  London penetration depth increases as T -> Tc (superconductivity weakens at surface).
##END##

Flux Quantization in Superconductors:
  Magnetic flux through a superconducting ring is QUANTIZED.
  Flux quantum (fluxon): phi_0 = h/(2e) = 2.07 x 10^-15 Wb (Weber).
  The factor 2e confirms carriers are Cooper PAIRS (charge = 2e).
  Flux through any superconducting loop = n * phi_0 (integer multiples only).
  In Type II vortex state: each vortex (flux tube) carries exactly ONE flux quantum phi_0.
  Flux quantization is direct experimental evidence for Cooper pairing in BCS theory.

Einstein Coefficients and Relations:
  Einstein (1917) introduced three coefficients for atom-radiation interaction:
  A21 (spontaneous emission coefficient): probability per unit time of spontaneous emission.
  B12 (absorption coefficient): probability per unit time of stimulated absorption.
  B21 (stimulated emission coefficient): probability per unit time of stimulated emission.
  Einstein relations: B12 = B21 (absorption and stimulated emission equally probable).
    A21 / B21 = 8*pi*h*nu^3 / c^3.
  For laser: need stimulated emission >> spontaneous emission.
  At thermal equilibrium: stimulated emission << absorption (normal state; lasers cannot operate).
  Ratio of stimulated to spontaneous emission: proportional to spectral energy density u(nu).
  Key insight: stimulated photon is IDENTICAL to incident photon (same phase, frequency, direction).

Semiconductor Laser (Laser Diode):
  Active medium: p-n junction of direct band-gap semiconductor (GaAs, InGaAsP).
  Pumping: forward bias voltage injects electrons (into p-side) and holes (into n-side).
  Population inversion: at junction, conduction band electrons recombine with valence band holes.
  Photon emission at wavelength: lambda = hc/Eg (depends on band gap energy).
  Resonator: cleaved facets of the semiconductor crystal act as partially reflective mirrors.
  LASER vs LED:
    LED: spontaneous emission; incoherent; wide spectral width; lower intensity.
    Laser diode: stimulated emission; coherent; narrow linewidth; high intensity; directional.
  Advantages: tiny size, low cost, direct electrical pumping, wavelength tunable by material.
  Applications: DVD/CD players, fiber optic communication (1300nm, 1550nm), laser printers,
    barcode scanners, laser pointers.
  GaAs laser: lambda ~870 nm. InGaAsP: lambda ~1300-1550 nm (telecom window).

Normal to a Surface and Angle Between Surfaces:
  UNIT NORMAL to surface phi(x,y,z) = c:
    n_hat = grad(phi) / |grad(phi)|
  NORMAL LINE at point P: passes through P in direction of grad(phi).
  TANGENT PLANE at point P to surface phi=c:
    dphi/dx*(x-x0) + dphi/dy*(y-y0) + dphi/dz*(z-z0) = 0
  ANGLE between two surfaces phi1=c1 and phi2=c2 at point of intersection:
    cos(theta) = (grad(phi1) . grad(phi2)) / (|grad(phi1)| * |grad(phi2)|)
  Surfaces are ORTHOGONAL if grad(phi1) . grad(phi2) = 0 at intersection.
  Example: Find normal to x^2+y^2+z^2=14 at (1,2,3):
    grad(phi) = 2xi + 2yj + 2zk = 2i+4j+6k at (1,2,3).

Important Vector Identities:
  Let phi, psi be scalar fields; A, B be vector fields:
  1. div(phi*A) = phi*div(A) + A.grad(phi)
  2. curl(phi*A) = phi*curl(A) + grad(phi) x A
  3. div(A x B) = B.curl(A) - A.curl(B)
  4. curl(curl(A)) = grad(div(A)) - nabla^2(A)
  5. grad(A.B) = (A.nabla)B + (B.nabla)A + A x curl(B) + B x curl(A)
  6. curl(grad(phi)) = 0  [curl of gradient is always zero]
  7. div(curl(A)) = 0     [divergence of curl is always zero]
  8. nabla^2(phi) = div(grad(phi)) = d^2phi/dx^2 + d^2phi/dy^2 + d^2phi/dz^2
  A vector field F is CONSERVATIVE if: curl(F) = 0 and F = grad(phi).
  A vector field F is SOLENOIDAL if: div(F) = 0.

Conservative Vector Field and Path Independence:
  A vector field F is CONSERVATIVE if the line integral is PATH INDEPENDENT.
  integral_C F.dr depends only on endpoints, not on the path C.
  Conditions for F = Pi + Qj + Rk to be conservative:
    curl(F) = 0, i.e., dR/dy=dQ/dz, dP/dz=dR/dx, dQ/dx=dP/dy.
  If F is conservative: F = grad(phi) for some scalar potential phi.
  Work done: W = integral_C F.dr = phi(B) - phi(A) (independent of path).
  For a CLOSED path: oint_C F.dr = 0 if F is conservative.
  Finding potential phi: integrate each component.
    dphi/dx = P, dphi/dy = Q, dphi/dz = R.
  Example: F = 2xy i + x^2 j is conservative.
    phi = x^2*y + C. Work from (0,0) to (2,3): phi(2,3)-phi(0,0) = 4*3 = 12.

Evaluating Line Integrals - Method and Examples:
  To evaluate integral_C F.dr along curve C:
  Step 1: Parametrize C: x=x(t), y=y(t), z=z(t), t from a to b.
  Step 2: Compute dx=x'(t)dt, dy=y'(t)dt, dz=z'(t)dt.
  Step 3: Substitute into integral and evaluate as ordinary integral in t.
  Example: F = x i + y j, C: y=x^2 from (0,0) to (1,1).
    Parametrize: x=t, y=t^2, t: 0->1. dx=dt, dy=2t dt.
    W = integral_0^1 (t*1 + t^2*2t) dt = integral_0^1 (t + 2t^3) dt = [t^2/2 + t^4/2]_0^1 = 1.
  For a straight line from A(x1,y1) to B(x2,y2):
    x = x1 + (x2-x1)*t, y = y1 + (y2-y1)*t, t: 0->1.
  If F is conservative: W = phi(B) - phi(A) (much easier).

Exact Differential Equations:
  Form: M(x,y)dx + N(x,y)dy = 0.
  Condition for exactness: dM/dy = dN/dx.
  If exact: solution is F(x,y) = C where dF/dx = M and dF/dy = N.
  Method: integrate M w.r.t. x, then differentiate w.r.t. y and compare with N.
  If NOT exact: use an INTEGRATING FACTOR mu(x) or mu(y) to make it exact.
    If (dM/dy - dN/dx)/N depends only on x: mu = exp(integral of that function dx).
    If (dN/dx - dM/dy)/M depends only on y: mu = exp(integral of that function dy).
  After multiplying by mu, the equation becomes exact; solve as above.
  Example: (2xy + y^2)dx + (x^2 + 2xy)dy = 0.
    dM/dy = 2x+2y = dN/dx. Exact! F = x^2*y + xy^2. Solution: x^2*y + xy^2 = C.

Spring-Mass System (SHM and Damping):
  Undamped free vibration: m*x'' + kx = 0; natural frequency w_n = sqrt(k/m).
    Solution: x = A*cos(w_n*t) + B*sin(w_n*t). No energy loss.
  Damped free vibration: m*x'' + c*x' + kx = 0.
    Auxiliary equation: m*r^2 + c*r + k = 0.
    Discriminant D = c^2 - 4mk.
    D > 0: Overdamped (two real roots); x decays without oscillation.
    D = 0: Critically damped (repeated root); fastest decay without oscillation.
    D < 0: Underdamped (complex roots); oscillates with decaying amplitude.
    Damping ratio: zeta = c/(2*sqrt(mk)).
  Forced vibration: m*x'' + c*x' + kx = F_0*cos(w*t).
    Particular integral gives steady-state response.
    RESONANCE: when forcing frequency w = natural frequency w_n; amplitude -> infinity (undamped).
##END##

Selection Sort in C:
  Find the minimum element in unsorted portion and place it at the beginning.
  Algorithm: n-1 passes; each pass selects minimum and swaps to correct position.
    for (i=0; i<n-1; i++) {
        min_idx = i;
        for (j=i+1; j<n; j++)
            if (a[j] < a[min_idx]) min_idx = j;
        if (min_idx != i) {
            temp=a[min_idx]; a[min_idx]=a[i]; a[i]=temp;
        }
    }
  Time complexity: O(n^2) always (always does n*(n-1)/2 comparisons).
  Space complexity: O(1) (in-place).
  Advantage over bubble sort: fewer swaps (at most n-1 swaps vs O(n^2) for bubble).
  After pass i: elements a[0..i] are in their final sorted position.
##END##

Binary Search in C:
  Requires SORTED array. Repeatedly halves the search space.

    while (low <= high) {
        mid = (low + high) / 2;
        if (a[mid] == key) { printf("Found at %d", mid); return; }
        else if (a[mid] < key) low = mid + 1;
        else high = mid - 1;
    }
    printf("Not found");

  vs Linear search: binary O(log n) vs linear O(n). Much faster for large arrays.

    union Data d;
    d.i = 10;           -- valid: stores 10
    d.f = 3.14;         -- overwrites d.i; now d.i is invalid
    printf("%f", d.f);  -- prints 3.14
  sizeof(union Data) = 20 (size of char str[20], the largest member).
  STRUCT vs UNION:
    Struct: all members have separate memory; total size = sum of members.
    Union: all members share memory; size = largest member; one valid at a time.
  Use case: when only ONE member will be active at a time (saves memory).
    Example: variant data type, network protocol headers.
##END##

Pointers in C - Definition and Declaration:
  A pointer is a variable that stores the MEMORY ADDRESS of another variable.

Pointer Arithmetic:
  Pointers support arithmetic operations: ++, --, +, -.
  When you add 1 to a pointer, it advances by sizeof(data_type) bytes.
    int a[5] = {10,20,30,40,50};

void Pointer and Generic Pointers:

Dynamic Memory Allocation in C:
  Static allocation: size fixed at COMPILE TIME (int a[100]).
  Dynamic allocation: size determined at RUNTIME; memory from HEAP.
  Four functions in <stdlib.h>:
  malloc(size): allocates 'size' bytes; returns void*; contents UNINITIALIZED (garbage).
    int *p = (int*)malloc(n * sizeof(int));
    if (p == NULL) { printf("Allocation failed"); exit(1); }
  calloc(n, size): allocates n*size bytes; ZERO-INITIALIZES all bytes.
    int *p = (int*)calloc(n, sizeof(int));
  realloc(ptr, new_size): resizes previously allocated block.
    p = (int*)realloc(p, new_n * sizeof(int));
    Returns new pointer (may be different address); old pointer invalid after call.
  free(ptr): releases heap memory back to OS.
    free(p); p = NULL;  -- set to NULL after free to avoid dangling pointer.
  ALWAYS check if malloc/calloc returns NULL (allocation failure).
  ALWAYS free every malloc/calloc to avoid memory leaks.
##END##

malloc vs calloc - Differences:
  FEATURE          | malloc                  | calloc
  Syntax           | malloc(size)            | calloc(n, size)
  Arguments        | 1 (total bytes)         | 2 (count, element size)
  Initialization   | NO (garbage values)     | YES (all bytes set to 0)
  Return type      | void*                   | void*
  Speed            | Slightly faster         | Slightly slower (zeroing takes time)
  Usage            | When initialization     | When you need zeroed memory
                   | not needed              | (counts, flags, numeric arrays)

  Both return NULL on failure. Both require free().
  Example:
    int *a = (int*)malloc(5 * sizeof(int));  -- a[0]..a[4] have garbage
    int *b = (int*)calloc(5, sizeof(int));   -- b[0]..b[4] are all 0
  realloc can be used with memory from either malloc or calloc.
##END##

Memory Leak and Dangling Pointer:
  MEMORY LEAK: allocated memory is never freed; program keeps consuming heap memory.
    void func() { int *p = malloc(100); /* no free */ }  -- leak on every call!
    Symptoms: program slows down, crashes after long run, system runs out of memory.
    Fix: every malloc/calloc must have a matching free().
  DANGLING POINTER: pointer that points to already-freed memory.
    int *p = malloc(sizeof(int));
    free(p);
    *p = 10;  -- UNDEFINED BEHAVIOUR! Dangling pointer dereference.
    Fix: set pointer to NULL after free: free(p); p = NULL;
    Then check: if (p != NULL) before using.
  DOUBLE FREE: calling free() twice on same pointer -> undefined behaviour.
    Fix: set p = NULL after first free; free(NULL) is safe and does nothing.
  NULL POINTER DEREFERENCE: *p when p is NULL -> crash (segmentation fault).
    Fix: always check if malloc returned NULL before using pointer.
  Good practice: initialize ALL pointers to NULL; check before use; free and null.
##END##

Linked List - Definition and Node Structure:
  A linked list is a dynamic data structure where elements (nodes) are stored in
  non-contiguous memory and linked using POINTERS.
  Each NODE contains:
    1. DATA field: stores actual data.
    2. NEXT pointer: stores address of the NEXT node (NULL for last node).
  Node structure in C:
    struct Node {
        int data;
        struct Node *next;   -- self-referential structure
    };
  HEAD pointer: points to the FIRST node; if HEAD == NULL, list is empty.
  TYPES of linked lists:
    1. Singly linked list: each node points to next only (one direction).
    2. Doubly linked list: each node has next AND prev pointers (both directions).
    3. Circular linked list: last node points back to first node.
  Linked list vs Array:
    Array: fixed size; random access O(1); insertion/deletion O(n) (shifting).
    Linked list: dynamic size; sequential access O(n); insertion/deletion O(1) at known position.
##END##

Singly Linked List - Insert at Beginning:
  To insert a new node at the BEGINNING of the list:
    struct Node* insert_front(struct Node* head, int data) {
        struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
        newNode->data = data;
        newNode->next = head;    -- new node points to current head
        head = newNode;          -- head now points to new node
        return head;
    }
  Steps: (1) Create new node with malloc. (2) Set its data. (3) Point new->next to old head. (4) Update head to new node.
  Time complexity: O(1) — insert at beginning is constant time.
  DISPLAY (traversal):
    struct Node* temp = head;
    while (temp != NULL) { printf("%d -> ", temp->data); temp = temp->next; }
    printf("NULL\n");

#UNIT=5
#KEYS=singly linked list insert end append last node NULL update
Singly Linked List - Insert at End:
  To insert a new node at the END of the list:
    struct Node* insert_end(struct Node* head, int data) {
        struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
        newNode->data = data;
        newNode->next = NULL;
        if (head == NULL) return newNode;  -- empty list
        struct Node* temp = head;
        while (temp->next != NULL) temp = temp->next;  -- traverse to last node
        temp->next = newNode;    -- link last node to new node
        return head;
    }
  Steps: (1) Create new node. (2) Set next=NULL. (3) Traverse to last node. (4) Link last->next to new node.
  Time complexity: O(n) — must traverse entire list to find end.
  Optimization: keep a TAIL pointer to avoid traversal; then O(1).
##END##

#TOPIC=cprog
#UNIT=5
#KEYS=singly linked list delete node beginning end middle free memory
Singly Linked List - Deletion:
  DELETE from BEGINNING:
    struct Node* delete_front(struct Node* head) {
        if (head == NULL) return NULL;
        struct Node* temp = head;
        head = head->next;
        free(temp);
        return head;
    }
  DELETE from END:
    Traverse to second-last node. Set its next = NULL. Free last node.
  DELETE a specific value:
    struct Node* delete_val(struct Node* head, int val) {
        if (head == NULL) return NULL;
        if (head->data == val) { struct Node* t=head; head=head->next; free(t); return head; }
        struct Node* prev = head, *curr = head->next;
        while (curr != NULL && curr->data != val) { prev=curr; curr=curr->next; }
        if (curr != NULL) { prev->next = curr->next; free(curr); }
        return head;
    }
  ALWAYS free the deleted node to prevent memory leak.
  Time complexity: O(1) at beginning; O(n) at end or for a specific value.
##END##

Doubly Linked List:
  Each node has TWO pointers: prev (to previous node) and next (to next node).
  Node structure:
    struct DNode {
        int data;
        struct DNode *prev;
        struct DNode *next;
    };
  HEAD->prev = NULL; TAIL->next = NULL.
  Advantages over singly linked list:
    - Can traverse in BOTH directions (forward and backward).
    - Deletion is O(1) if pointer to node is known (no need to find prev node).
    - Can insert before a given node easily.
  Disadvantages: extra memory for prev pointer per node.
  INSERT at beginning:
    newNode->next = head; newNode->prev = NULL;
    if (head != NULL) head->prev = newNode;
    head = newNode;
  DELETE a node:
    if (node->prev) node->prev->next = node->next;
    if (node->next) node->next->prev = node->prev;
    free(node);
##END##

Circular Linked List:
  In a circular linked list, the LAST node's next pointer points back to the FIRST node.
  Singly circular: last->next = head (no NULL in the list).
  Doubly circular: last->next = head AND head->prev = last.
  ADVANTAGES:
    - Can start traversal from ANY node and reach all other nodes.
    - Useful for applications that require cyclic iteration (round-robin scheduling, music playlist).
  TRAVERSAL: stop when you return to starting node (not NULL check).
    struct Node* temp = head;
    do { printf("%d ", temp->data); temp = temp->next; } while (temp != head);
  INSERT at beginning: new->next = head; find last node; last->next = new; head = new.
  Detecting circular list (Floyd's algorithm): slow and fast pointer; if they meet, loop exists.
##END##

File Positioning Functions:
  fseek(fp, offset, origin): moves file position pointer.
    origin values:
      SEEK_SET (0): from beginning of file.
      SEEK_CUR (1): from current position.
      SEEK_END (2): from end of file.
    Examples:
      fseek(fp, 0, SEEK_SET);   -- go to beginning
      fseek(fp, 0, SEEK_END);   -- go to end

  ftell(fp): returns current position (offset from beginning in bytes).
    long pos = ftell(fp);
  rewind(fp): equivalent to fseek(fp, 0, SEEK_SET); resets to beginning.
  Random access: use fseek to jump to any record directly.
    Particularly useful for BINARY files with fixed-size records.

Binary File Operations - fread and fwrite:
